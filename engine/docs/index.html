Language Page: House Rules for a Bits‑Native System

A concise, readable style guide for working with A,U,P,E,Δ,I,R,T,M — designed like a newspaper for skimmability.

Dateline — Today

A Bits are the lingua franca. Every action is gated: Ask–Act (A=1 ∧ P=1 ∧ Δ=0) and Evidence (U≥τ → verify‑first). Keep prose minimal, show receipts, and ship tiny diffs.

Speak in artifacts, not promises: a diff, a receipt, a golden comparison. When in doubt, compress to intent triad (goal, constraints, evidence).

Style

  • Short sentences, active voice, concrete nouns.
  • Prefer bullets over paragraphs; one idea per line.
  • Name bits and gates explicitly when relevant.

Bits Cheat Sheet

  • A Alignment • U Uncertainty • P Permission • E Error
  • Δ Drift • I Interrupt • R Recovery • T Trust • M Meta

Control Laws

  1. Ask–Act: only act when A=1 ∧ P=1 ∧ Δ=0.
  2. Evidence: if U≥τ, verify‑first (dry‑run + refs).
  3. CAPS: network/file_write/identity require P=1.

Rubrics

Score results with R = 2·pass − 0.1·time − 0.1·cost − 0.05·mdl. Promote what reliably improves R on goldens.

Links

  • Meta step: POST /meta/run → β plan + γ config pick.
  • Golden validation: POST /validate_golden → pass/fail + bits.
  • Chat loop: POST /users/:id/chat → reply + bits + beacons.
Closure beats scope: plan → act → verify → stop. If it’s not improving R or bits, it’s likely noise.
